Prototype Pattern

Definition

What is prototype pattern ?

Prototype pattern are memory efficient since every public functions and varibales are declared in the prototype of the function therefore, whenever instance are created for that function every instance will not have copy of those functions and variables which are in prototype. Rather those instances will have reference to the variables and functions hence it is memory efficient.

When We need it ?

When we want to create a class and whose various instances has to be used in many places. For example if we create a Animal Class and we can create many concreate animals later.

What is disadvantages of prototype pattern

In Module Pattern public function can access private functions. However in prototype pattern public function declared in prototype object can not access any private functions or variables.

Example ( Please check console log for output of below example )

 
                function Logger() {
                }
                
                /** private variable */
                Logger.prototype.version = 'Logger Version 2.3: ';
                
                /** public function */
                Logger.prototype.info = function (msg) {
                    console.info(this.version + msg);
                }
                
                /** public function */
                Logger.prototype.debug = function (msg) {
                    console.debug(this.version + msg);
                }
                
                
                var log = new Logger(); // we have to use new to create instance
                log.info('This is info message');
                log.info('This is debug message');
